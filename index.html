Multiplexer & Demultiplexer

Memahami konsep "Many-to-One" dan "One-to-Many" dalam rangkaian logika digital serta aplikasinya pada sistem komunikasi.

Pendahuluan

Dalam sistem digital modern, efisiensi jalur komunikasi adalah kunci. Seringkali kita perlu mengirimkan data dari banyak sumber melalui satu kabel saja, atau sebaliknya, membagikan data dari satu kabel ke banyak tujuan.

Disinilah peran Multiplexer (Mux) dan Demultiplexer (Demux). Keduanya bekerja berpasangan layaknya pengatur lalu lintas data yang canggih.

1. Multiplexer (Mux)

Multiplexer adalah rangkaian logika yang berfungsi sebagai Data Selector (Pemilih Data). Ia menerima banyak input data, namun hanya mengizinkan satu data untuk lewat ke output dalam satu waktu.

  • Prinsip: Many-to-One (Banyak ke Satu).
  • Cara Kerja: Sinyal pada kaki Selector menentukan input mana yang terhubung ke output.
  • Rumus: Input = 2n, dimana n adalah jumlah selector.
  • Aplikasi: Penghematan jalur transmisi (telepon, internet).

2. Demultiplexer (Demux)

Demultiplexer adalah kebalikan dari Mux. Ia berfungsi sebagai Data Distributor. Ia menerima data dari satu jalur input dan menyalurkannya ke salah satu dari banyak jalur output.

  • Prinsip: One-to-Many (Satu ke Banyak).
  • Cara Kerja: Sinyal selector mengarahkan data input ke pintu output yang spesifik.
  • Relasi: Sering digabungkan dengan decoder.

Perbedaan Utama

Fitur Multiplexer (Mux) Demultiplexer (Demux)
Fungsi Utama Memilih Data (Selector) Membagi Data (Distributor)
Arah Data Banyak Input → 1 Output 1 Input → Banyak Output
Lokasi Sisi Pengirim (Transmitter) Sisi Penerima (Receiver)
